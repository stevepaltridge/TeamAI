## 2. Project Scope & Objectives

### Problem Statement

Over several years, the MortgageTech Azure subscription accumulated:
- **36 storage accounts** — the majority auto-generated by VM diagnostics, boot diagnostics, and now-decommissioned VMs
- **8 managed disks** — including 6 unattached disks from previously deleted or decommissioned VMs
- **3 snapshots** — orphaned point-in-time captures no longer associated with active workloads
- **4 virtual machines** — 3 of which were deallocated (stopped) and incurring ongoing disk storage costs
- Dozens of orphaned networking resources (NICs, NSGs, Public IPs) left behind after prior VM deletions

### Objectives

1. **Archive all recoverable data** before deleting any resource
2. **Verify every archive** at the byte level (MD5 hash or exact size match)
3. **Delete only confirmed-safe resources** — never delete anything in active use
4. **Protect critical resources** with Azure resource locks
5. **Document everything** for audit trail and recovery purposes

### Constraints / Rules

- **Kris's VM ecosystem** (Dev-Kris-VM02 and all associated resources) was explicitly **OFF-LIMITS** — never modified, never stopped
- **Production workloads** (encompasssyncfunc, workbooksfxstorage, static web apps) were never touched
- **Cloud Shell storage** (cs7d67f85c…, mtgshellstorage) was preserved
- All operations were performed with explicit verification at each step

## 9. Recovery Procedures

If any archived data needs to be restored, the following procedures are available.

### 9.1 Restore a VHD or Disk from Archive

```powershell
# 1. List blobs in a container
az storage blob list --account-name mtgtecharchive --container-name <container> --output table

# 2. Generate a download SAS for the blob
az storage blob generate-sas --account-name mtgtecharchive --container-name <container> --name <blob-name> --permissions r --expiry <date> --output tsv

# 3. Download using azcopy
azcopy copy "https://mtgtecharchive.blob.core.windows.net/<container>/<blob>?<SAS>" "C:\local\path\"

# 4. Create a new managed disk from the downloaded VHD
az disk create --resource-group <RG> --name <new-disk-name> --source "https://mtgtecharchive.blob.core.windows.net/<container>/<blob>?<SAS>" --size-gb <size>
```

### 9.2 Recover from Soft-Delete (Accidental Deletion)

```powershell
# List deleted blobs
az storage blob list --account-name mtgtecharchive --container-name <container> --include d --output table

# Undelete a specific blob
az storage blob undelete --account-name mtgtecharchive --container-name <container> --name <blob-name>

# List deleted containers
az storage container list --account-name mtgtecharchive --include-deleted --output table
```

### 9.3 Verify Archive Integrity

```powershell
# Check soft-delete settings
az storage blob service-properties delete-policy show --account-name mtgtecharchive

# Check container soft-delete
az storage account blob-service-properties show --account-name mtgtecharchive --resource-group mtgtecharchive --query "containerDeleteRetentionPolicy"

# Check resource lock
az lock list --resource-group mtgtecharchive --output table

# List all blobs with sizes
az storage blob list --account-name mtgtecharchive --container-name vhds --output table --query "[].{Name:name, Size:properties.contentLength}"
az storage blob list --account-name mtgtecharchive --container-name disks --output table --query "[].{Name:name, Size:properties.contentLength}"
az storage blob list --account-name mtgtecharchive --container-name snapshots --output table --query "[].{Name:name, Size:properties.contentLength}"
```
