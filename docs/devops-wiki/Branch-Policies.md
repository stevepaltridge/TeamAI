# Branch Policies

Rules for how code and knowledge flow into the TeamAI repository.

## Current Policy: Direct Push to Main

**Status:** Direct pushes to `main` are currently allowed for all team members.

This is appropriate for our team size (8 people) and content type (mostly knowledge articles and configuration). If the team grows or we start making breaking changes, Steve will enable PR-based reviews.

## Recommended Practices (Even Without Enforced PRs)

1. **Pull before you push.** Always run `git pull` before pushing. The workspace does this automatically on open, but do it again if you've been working offline.
2. **Small, focused commits.** One topic per commit. Don't bundle unrelated changes.
3. **Descriptive commit messages.** Say what changed and why:
   - Good: `Add knowledge article: TRID disclosure timing rules`
   - Bad: `update`
4. **Don't force-push.** Never use `git push --force`. If you have conflicts, ask Steve.
5. **Don't edit auto-generated files.** The `knowledge/encompass-settings/` folder is generated by scripts. Edits there will be overwritten.

## When to Enable PR Reviews

Steve should enable branch policies on Azure DevOps when any of these become true:

| Trigger | Action |
|---------|--------|
| Team exceeds 10 contributors | Require 1 reviewer for PRs to main |
| Breaking changes to `.github/copilot-instructions.md` | Require Steve's approval |
| Scripts or API changes | Require Kris (developer) review |
| Conflicting edits become frequent | Enable merge queue |

## How to Enable (Steve — Azure DevOps)

1. Go to **Project Settings** > **Repositories** > **TeamAI** > **Policies**
2. Under **Branch Policies** for `main`:
   - **Require a minimum number of reviewers:** 1
   - **Check for linked work items:** Optional
   - **Check for comment resolution:** Required
3. Add **Build Validation** if CI pipeline exists
4. Save

## Conflict Resolution

If `git push` fails with merge conflicts:

1. `git pull --rebase` (auto-rebases your changes on top of latest main)
2. If conflicts appear, open the conflicting files, resolve, then:
   ```powershell
   git add .
   git rebase --continue
   git push
   ```
3. If stuck, call Steve. Don't force-push.
